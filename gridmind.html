01

The problem

A compliance engineer asked to demonstrate conformance with one NERC requirement often has to open several 30–60 page standards, cross-reference them, check which revision is current, and judge whether each clause is mandatory ('shall') or advisory ('should') — today done with Ctrl-F over PDFs. Generic RAG makes it worse: it hallucinates citations, ignores document versions, and treats binding and advisory language identically. GridMind targets exactly that gap.

Architecture

An offline indexer builds the corpus; an event-driven online pipeline ingests it idempotently; a hybrid retriever serves it. The data path below is the indexing-and-serving spine.

Data path

NERC / FERC docsSection-aware chunkerDistilBERT · obligation tagEmbed · MiniLMAurora · pgvectorHybrid retrievalFastAPI / Streamlit
State bridge — DynamoDB checksums skip unchanged revisions (idempotent re-ingest)
Orchestration — Step Functions DistributedMap fans out ~40 Lambdas
Hybrid index — pgvector HNSW for semantics + tsvector GIN for keywords
Cross-refs — NetworkX citation graph expands related standards

▸ AWS serverless throughout (Lambda, Step Functions, Aurora) — designed to run under ~$5/month

A monthly offline indexer scrapes standards, chunks them along requirement boundaries, tags each chunk's obligation strength, embeds it, and publishes a bundle to S3. An S3 event triggers the online Step Functions pipeline, which unpacks, checksums against DynamoDB to skip unchanged revisions, and upserts chunks into Aurora with pgvector. At query time a hybrid retriever scores candidates 60% semantic + 25% keyword + 10% freshness + 5% obligation, then expands across the citation graph.

03

Key decisions & tradeoffs

The design choices that make this more than a PDF chatbot.

Ingestion

Two pipelines + a DynamoDB checksum bridge

An offline indexer and an online ingestion pipeline share a DynamoDB state table keyed by standard and version. Unchanged revisions are skipped by checksum, so the monthly run doesn't re-embed the entire corpus. The ingestion is idempotent by construction — re-running is safe.

Chunking

Section-aware, not token-count

NERC standards are structured as Requirements (R1, R2.3, …). The chunker respects those boundaries so a chunk never splits mid-requirement. Retrieval then returns coherent obligations instead of fragments that lose their meaning at the cut point.

Ranking

Obligation-aware hybrid scoring

The retrieval contribution that separates this from generic RAG: each chunk is tagged shall / should / may / informational by a fine-tuned DistilBERT, and a binding 'shall' outranks a semantically similar 'may'. Mandatory beats advisory, folded into one score alongside semantic, keyword, and freshness signals.

Retrieval

Cross-reference graph expansion

Standards cite each other (CIP-013 points at CIP-005). A NetworkX citation graph lets a result expand to the standards it references, so a single question surfaces both the clause asked about and the definitions it depends on.

Versioning

Version-aware retrieval

Each document carries its effective date and whether it's been superseded. The retriever prefers the current revision, so you don't get a confident answer drawn from a retired version of a standard.
